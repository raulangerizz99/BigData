Almacenamiento y bases de datos NoSQL

El almacenamiento es un componente clave en Big Data, ya que los datos generados son masivos, diversos y requieren tecnologías eficientes para su gestión. En este contexto, las bases de datos NoSQL han surgido como una alternativa flexible a las bases de datos tradicionales (SQL), permitiendo manejar grandes volúmenes de información distribuida.


1.Almacenamiento en Big Data
El almacenamiento de Big Data se basa en sistemas distribuidos, diseñados para manejar grandes volúmenes de datos de manera eficiente. Existen diferentes tipos de soluciones de almacenamiento:

a) Sistemas de Archivos Distribuidos (HDFS, Amazon S3, Google Cloud Storage)
HDFS (Hadoop Distributed File System): Almacena datos en múltiples nodos dentro de un clúster y permite procesamiento paralelo con Hadoop.
-Amazon S3 y Google Cloud Storage: Soluciones en la nube que ofrecen escalabilidad y alta disponibilidad para datos no estructurados.
-Ejemplo: Empresas de análisis de datos usan HDFS para almacenar logs de servidores y datos de sensores IoT.

b) Bases de Datos NoSQL
-Las bases de datos NoSQL (Not Only SQL) son sistemas diseñados para manejar datos no estructurados o semiestructurados, escalando horizontalmente en múltiples servidores. Se utilizan en aplicaciones de Big Data debido a su flexibilidad y rendimiento.


2.Características de las Bases de Datos NoSQL
✅ Escalabilidad horizontal: Pueden crecer distribuyendo datos en múltiples servidores.
✅ Alta disponibilidad: Son tolerantes a fallos, asegurando que los datos siempre estén accesibles.
✅ Flexibilidad en los datos: No requieren un esquema rígido como en SQL.
✅ Rendimiento optimizado: Pueden manejar grandes volúmenes de lectura/escritura en tiempo real.


3.Tipos de Bases de Datos NoSQL y Ejemplos
a) Bases de Datos Clave-Valor 
-Almacenan datos como pares clave-valor, similar a un diccionario.
-Casos de uso: Caché, sesiones de usuario, configuración de aplicaciones.
-Ejemplos:Redis: Rápida y utilizada para almacenamiento en memoria.
-DynamoDB (Amazon): Base de datos NoSQL escalable en la nube.

b) Bases de Datos de Documentos 
-Almacenan datos en formato JSON, BSON o XML, permitiendo consultas flexibles.
-Casos de uso: Aplicaciones web, comercio electrónico, almacenamiento de logs.
-Ejemplos: MongoDB: Base de datos NoSQL más popular para aplicaciones escalables.

c) Bases de Datos de Columnas Anchas 
-Diseñadas para almacenar y recuperar grandes volúmenes de datos estructurados en columnas en lugar de filas.
-Casos de uso: Big Data, análisis de logs, telecomunicaciones.
-Ejemplos:Apache Cassandra: Base de datos altamente escalable utilizada por Facebook.
-HBase: Base de datos basada en HDFS, optimizada para Hadoop.

d) Bases de Datos de Grafos 
-Modelan relaciones entre datos en forma de nodos y aristas.
-Casos de uso: Redes sociales, análisis de fraudes, motores de recomendación.
-Ejemplos: Neo4j: Base de datos de grafos utilizada en redes sociales y detección de fraudes.
-ArangoDB: Permite consultas híbridas de grafos y documentos.

LinkedIn y Facebook usan bases de datos de grafos para modelar relaciones entre usuarios y recomendar conexiones.
