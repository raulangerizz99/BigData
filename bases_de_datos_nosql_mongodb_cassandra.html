Bases de datos NoSQL: MongoDB, Cassandra

1.¿Qué son las Bases de Datos NoSQL?
Las bases de datos NoSQL (Not Only SQL) son una alternativa a las bases de datos relacionales tradicionales. Están diseñadas para manejar grandes volúmenes de datos no estructurados o semiestructurados, permitiendo mayor escalabilidad, flexibilidad y velocidad en el procesamiento.

2.Tipos de Bases de Datos NoSQL
Existen cuatro tipos principales de bases de datos NoSQL, cada una optimizada para casos de uso específicos.

a) Bases de Datos Documentales
📌 Almacenan datos en documentos JSON o BSON en lugar de tablas.
📌 Son flexibles y permiten estructuras dinámicas.

-Ejemplos:
✅ MongoDB
✅ CouchDB
✅ Firebase Firestore

-Casos de uso: Aplicaciones web y móviles, almacenamiento de contenido dinámico, sistemas de gestión de datos flexibles

b) Bases de Datos Clave-Valor
📌 Funcionan como un diccionario donde cada dato se almacena como un par clave-valor.
📌 Son extremadamente rápidas y eficientes.

-Ejemplos:
✅ Redis
✅ Amazon DynamoDB
✅ Memcached

-Casos de uso: Caché de datos, administración de sesiones de usuario, juegos en línea y sistemas en tiempo real

c) Bases de Datos Columnar
📌 Almacenan los datos en columnas en lugar de filas, optimizando consultas masivas.
📌 Son ideales para Big Data y análisis en tiempo real.

 -Ejemplos:
✅ Apache Cassandra
✅ HBase
✅ Google Bigtable

-Casos de uso: Almacenamiento de grandes volúmenes de dato, procesamiento de logs y métricas, aplicaciones de inteligencia de negocios

d) Bases de Datos de Grafos
📌 Diseñadas para almacenar y analizar relaciones complejas entre datos.
📌 Son utilizadas en redes sociales, motores de recomendación y análisis de fraudes.

-Ejemplos:
✅ Neo4j
✅ ArangoDB
✅ Amazon Neptune

-Casos de uso: Redes sociales, motores de recomendación, detección de fraudes


3.Principales Bases de Datos NoSQL
🔹 MongoDB (Documental)
✅ Modelo basado en documentos JSON/BSON.
✅ Usado por empresas como eBay y Uber.
✅ Consultas avanzadas y escalabilidad horizontal.

🔹 Apache Cassandra (Columnar)
✅ Distribución peer-to-peer sin puntos únicos de fallo.
✅ Utilizado por Facebook, Netflix y Twitter.
✅ Optimizado para Big Data y análisis en tiempo real.

🔹 Redis (Clave-Valor)
✅ Base de datos en memoria para máxima velocidad.
✅ Ideal para caché y almacenamiento de sesiones.
✅ Utilizado por GitHub, Instagram y StackOverflow.

🔹 Neo4j (Grafos)
✅ Especializado en relaciones entre datos.
✅ Usado por LinkedIn y eBay para análisis de redes.
✅ Soporta consultas avanzadas con Cypher Query Language.


4.Ventajas y Desventajas de NoSQL
📌 Ventajas
✔️ Escalabilidad horizontal (se expanden fácilmente con más servidores).
✔️ Flexibilidad (no requieren esquemas rígidos como SQL).
✔️ Alto rendimiento (rápidas en lectura/escritura de datos).
✔️ Tolerancia a fallos (distribución de datos en múltiples nodos).

📌 Desventajas
❌ No todas soportan transacciones ACID (algunas usan modelo BASE).
❌ Pueden ser complejas de administrar en grandes entornos.
❌ No siempre son la mejor opción para datos altamente relacionales.
