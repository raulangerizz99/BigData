Hadoop y MapReduce

1.¿Qué es Apache Hadoop? 🐘
Apache Hadoop es un framework de código abierto diseñado para el almacenamiento y procesamiento distribuidor de grandes volúmenes de datos en un clúster de computadoras. Se basa en un modelo escalable y tolerante a fallos, lo que lo convierte en una de las herramientas más utilizadas en Big Data.

-Características clave de Hadoop:
✅ Escalabilidad: Puede manejar petabytes de datos distribuidos en cientos o miles de servidores.
✅ Tolerancia a fallos: Si un nodo falla, el sistema redistribuye automáticamente los datos y tareas.
✅ Procesamiento paralelo: Usa el modelo MapReduce para ejecutar tareas en paralelo.
✅ Almacenamiento distribuido: Utiliza HDFS (Hadoop Distributed File System) para gestionar grandes volúmenes de datos.
✅ Código abierto: Permite personalización y es compatible con múltiples herramientas de análisis.


2.Componentes principales de Hadoop
a) HDFS (Hadoop Distributed File System) 🗄️
-Es el sistema de almacenamiento de Hadoop, diseñado para manejar grandes archivos de datos distribuidos en múltiples nodos.
-Usa un enfoque Maestro-Esclavo, donde un NameNode gestiona la estructura del sistema y los DataNodesalmacenan los bloques de datos.
-Ejemplo de uso: Empresas como Facebook y Yahoo almacenan registros de usuarios en HDFS para análisis posterior.
b) YARN (Yet Another Resource Negotiator) 🎛️
-Es el gestor de recursos que distribuye las tareas entre los nodos del clúster.
-Permite que múltiples aplicaciones (como Spark o MapReduce) compartan los recursos del sistema.
c) MapReduce ⚙️
-Es el modelo de programación que permite procesar grandes volúmenes de datos en paralelo.
-Se basa en dos fases:Map: Divide los datos en pequeñas tareas y las distribuye en múltiples nodos.
-Reduce: Recoge y combina los resultados del proceso Map para generar un resultado final.


3.¿Qué es MapReduce?
MapReduce es un modelo de programación distribuida desarrollado por Google y utilizado en Hadoop para procesar grandes volúmenes de datos en paralelo.

1️⃣ Fase Map (División y Asignación)

-Los datos de entrada se dividen en pequeños fragmentos.
-Se aplican funciones para transformar cada fragmento en pares clave-valor.
-Ejemplo: Si se analiza un conjunto de textos, la fase Map puede contar cuántas veces aparece cada palabra.

2️⃣ Fase Shuffle (Intercambio de datos)

-Se agrupan los datos con la misma clave para facilitar su procesamiento.
-Ejemplo: Todas las ocurrencias de la palabra "Big Data" se agrupan juntas.

3️⃣ Fase Reduce (Procesamiento y Resultado Final)

-Se aplican funciones para combinar los datos agrupados y generar un resultado final.
-Ejemplo: Se suman todas las ocurrencias de cada palabra para obtener un conteo total.
 

4.Casos de Uso de Hadoop y MapReduce
📌 Análisis de registros de servidores: Empresas como Yahoo y Facebook usan MapReduce para analizar millones de registros de actividad.

📌 Procesamiento de datos en el sector financiero: Bancos utilizan Hadoop para detectar fraudes en transacciones masivas.

📌 Genómica y salud: Hadoop se usa en la secuenciación del ADN para analizar grandes volúmenes de datos biomédicos.

📌 Motores de recomendación: Empresas como Netflix y Amazon emplean Hadoop para analizar el comportamiento de los usuarios y recomendar productos.
